# 运行与运维说明

## 文档用途

本文档记录服务的运行方式、配置来源、外部运行依赖与观测排障入口，用于帮助新会话快速把系统跑起来并定位常见运行问题。
默认适用于单仓库单服务的 Go 后端项目，也可作为仓库 Agent 的补齐指令。

## 补齐指引

- 优先记录当前仍然有效的运行方式和观测入口，不保留过期命令或旧环境说明。
- 重点写清“如何启动”“配置从哪里来”“依赖什么”“出问题先看哪里”。
- 如果项目存在多种运行方式，优先写默认主路径，再补充常见变体。
- 无法从仓库内确认的运维信息不要猜测，直接写 `待确认`。

## 优先信息来源

- `Makefile`、`magefile`、`Taskfile`、启动脚本与 `docker-compose` 文件
- `Dockerfile`、Kubernetes / Helm / 部署目录与 CI/CD 配置
- 配置样例、环境变量定义、配置加载代码与 `.env.example`
- 日志、指标、链路追踪、告警说明与排障文档

## 完成标准

- 读者能按文档说明找到服务启动方式、配置来源和外部依赖。
- 读者知道日志、指标或链路入口，以及常见故障先去哪里看。
- 文档聚焦运行上下文，没有混入系统职责边界或代码分层设计。

## 缺失信息写法

- 未确认的启动步骤、配置来源或观测入口写 `待确认`。
- 项目未使用的运行依赖或观测能力写 `不适用`。
- 仓库内信息彼此矛盾时写 `存在冲突，待核实`，并指出冲突文件。

## 建议阅读时机

- 在本地启动、联调、部署或排查运行异常前阅读。
- 在判断配置、基础设施依赖或监控告警入口时阅读。

## 适合写入

- 服务启动方式
- 配置来源与生效机制
- 外部运行依赖
- 部署与运行环境概况
- 日志、指标与链路入口
- 常见排障入口

## 不适合写入

- 系统职责边界、业务目标与非目标范围
- 代码内部结构、分层设计与模块职责
- 当前迭代状态、历史决策背景或未确认风险猜测
